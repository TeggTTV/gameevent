First, run the development server:

```bash

# or
yarn dev
# or
pnpm dev
# or

```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Environment Setup

Create a local environment file before running the app:

```bash
cp .env.example .env.local
```

Generate a strong secret and set it in `.env.local`:

```bash
openssl rand -hex 32
```

Set the generated value as `LOCAL_AUTH_SECRET`. In production, this variable is required and the server will not sign/verify tokens without it.

## Production Note

This MVP currently stores active game state in process memory. In production, the server will throw on startup unless you explicitly confirm single-instance deployment:

```bash
BUSYTHRIFT_SINGLE_INSTANCE=true
```

Do not run multiple app instances with this in-memory store. Use a shared datastore (for example Redis/Postgres) before horizontal scaling.
